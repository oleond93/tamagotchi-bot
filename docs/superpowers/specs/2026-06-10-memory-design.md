# Памʼять аватара (натхнення — Hermes Agent)

**Дата:** 2026-06-10
**Статус:** затверджено власником проєкту (повна автономія реалізації)

## Мета

Зараз кожне повідомлення **stateless**: LLM не бачить ні попередніх реплік, ні фактів про
власника. З Hermes Agent забираємо **механіку памʼяті** й адаптуємо під наш тамагочі —
у двох шарах:

1. **Короткострокова памʼять діалогу** — аватар памʼятає останні репліки поточної розмови
   (передаються в LLM як справжня історія чату).
2. **Довгострокова курована памʼять про власника** — аватар сам формує короткі факти
   («у власника іспит», «є кіт Мурчик», «любить каву»), зберігає їх і доречно згадує
   в чаті та в нагадуваннях.

Це робить звʼязок «живим», лишаючись у межах **$0 / free-tier** (без нових таблиць,
без зайвих викликів API).

## Ключові рішення

- **Зберігання** — у тому ж JSON-блобі `data` (поля `dialog`, `memories` у `Pet.toJSON()`).
  Жодних нових таблиць чи міграцій — сумісно з наявною архітектурою «один рядок на чат».
- **Захоплення фактів** — інлайн-тег `[[mem:факт]]` у кінці відповіді, рівно як уже зроблено
  для емоцій `[[emo:KEY:СИЛА]]`. **Нуль додаткових викликів API.** Періодичну LLM-рефлексію
  свідомо НЕ робимо (дорого на free-tier) — натомість дедуп/забування суто кодом.
- **Забування** — кап на кількість фактів; при переповненні витісняється найстаріший (за
  часом). Повтор близького факту не дублюється, а «підкріплює» наявний (оновлює свіжість).
  Тематично: аватар забуває давнє, памʼятає те, що згадуєш часто.
- **Стадії** — памʼять і діалог працюють лише **після вилуплення** (`!isEgg()`); захоплення
  фактів — лише поки `alive`. Яйце памʼяті не має.

## Модель даних (нове в `Pet`)

```jsonc
"dialog":   [ { "r": "u" | "a", "t": "<текст репліки>" }, ... ],  // вікно останніх реплік
"memories": [ { "t": "<короткий факт про власника>", "at": <unix-сек> }, ... ]
```

Константи (`pet.js`): `DIALOG_MAX=12` (≈6 реплік туди-сюди), `DIALOG_ENTRY_MAX=280`,
`MEMORY_MAX=30`, `MEMORY_ENTRY_MAX=160`, `MEMORY_PROMPT_MAX=12` (скільки фактів інжектимо
в промпт). `DIALOG_MAX`/`MEMORY_MAX` експортуються для тестів.

## Компоненти

### `pet.js` (чиста логіка)
- `rememberDialog(role, text)` — додає репліку у вікно, обрізає до `DIALOG_MAX`. No-op для яйця.
- `dialogMessages()` — `dialog` → `[{role:'user'|'assistant', content}]` для chat-API.
- `addMemory(text)` — нормалізує, дедуп (рівність або підрядок → підкріплення наявного,
  без дубля), обрізає, додає, витісняє найстаріший понад `MEMORY_MAX`. No-op для яйця/мертвого.
- `recentMemories(limit)` — найсвіжіші перші, обмежено.
- `memoryPrompt()` — блок «ЩО ТИ ПАМʼЯТАЄШ ПРО ВЛАСНИКА» для системного промпту (або "").
- `memoryCard()` — HTML-картка для `/memory` та кнопки (екранує текст факту).

### `brain.js` (LLM)
- `parseReply(raw)` → `{text, emotion, intensity, memory}` — вирізає **обидва** теги
  (`[[emo:…]]` і `[[mem:…]]`), бере останній кожного типу.
- `parseEmotionTag(raw)` — лишається (зворотна сумісність); тепер теж вирізає `[[mem:…]]`,
  щоб тег не протік у текст.
- `chat(...)` — приймає `history` і вставляє її між `system` і поточним `user`.
- `systemPrompt(...)` — додає `pet.memoryPrompt()` (згадування) і, лише в `reply` для
  не-яйця/живого, інструкцію-тег `[[mem:…]]` (захоплення).
- `reply(...)` — передає `history: pet.dialogMessages()`, повертає ще й `memory`.
- `nudge(...)` — у промпт додає підказку доречно згадати факт; памʼять уже в системному промпті.

### `index.js` (Worker)
- Вільний текст: після `brain.reply` → `applyEmotion`, `if (memory) addMemory(memory)`,
  `rememberDialog('u', text)`, `rememberDialog('a', answer)`, потім save.
- `/memory` — показує `memoryCard()`.
- Кнопка `🧠 Памʼять` на картці (лише не-яйце) + callback `mem`.
- `HELP`/`guide` згадують памʼять.

## Потік даних (вільний чат)

```
user msg → load pet → applyDecay → brain.reply(pet, msg)
                                      ├─ system: характер+стан+ПАМʼЯТЬ(факти)+емоції+інструкції-теги
                                      ├─ history: dialogMessages() (попередні репліки)
                                      └─ user: поточне повідомлення
        → parseReply → {text, emotion, intensity, memory}
        → applyEmotion · addMemory(memory) · rememberDialog(u)+rememberDialog(a)
        → send(text) → save
```

## Обробка помилок / межі
- Зіпсований/відсутній тег `[[mem:…]]` → `memory=null` (просто не запамʼятовуємо).
- Текст факту з `&<>` екранується в `memoryCard` (parse_mode=HTML).
- Кап і обрізання гарантують обмежений розмір блоба й промпту (важливо для D1 і токенів).
- Яйце/мертвий: захоплення вимкнене; згадування для яйця порожнє.

## Тести (`test/memory.test.mjs`, без фреймворку)
- `parseReply`: вирізає mem; mem+emo разом; немає mem → null; останній mem; обрізання; порожньо.
- `parseEmotionTag`: зворотна сумісність + вирізає протеклий mem.
- `addMemory`: додавання; дедуп (рівність/підрядок → підкріплення, без дубля, свіжість);
  кап і витіснення найстарішого; no-op яйце/мертвий; обрізання.
- `rememberDialog`/`dialogMessages`: пуш, обрізання вікна, мапінг ролей; no-op яйце.
- `recentMemories`, `memoryPrompt`, `memoryCard` (порожньо/список/екранування).
- round-trip `toJSON` → `new Pet(...)`: `dialog` і `memories` зберігаються.
