# Характер як «клас»: пасиви, синергія дій, фірмова здібність (Фаза 1)

**Дата:** 2026-06-10
**Статус:** затверджено власником (повна автономія). Фаза 1 із двох (Фаза 2 — гілчаста еволюція).

## Мета

Зараз характер (persona) і вдача (temperament) впливають **лише на тон** відповідей LLM —
нульова механічна вага. Як у RPG (класи), даємо характеру **механічну вагу**, щоб вибір
догляду (що ліпить характер) став вибором білда з трейд-офами.

> Розподіл осей: **Фаза 1 = характер** (persona). Вдача отримає велику механічну роль у
> **Фазі 2** (гілчаста еволюція за вдача × характер). Емоційну роль вдачі (множники
> `TEMPERAMENT_REACT`) не чіпаємо.

## Чотири механіки

1. **Пасив занепаду** — кожен архетип множить швидкість занепаду певних показників
   (`<1` = легше, `>1` = важче). «Конституція» класу.
2. **Синергія дій** — дія резонує (×1.4 на основний показник) або бекфайрить (×0.6 +
   `-5` настрою) залежно від характеру. Миттєвий механічний фідбек + флейвор у тості.
3. **Фірмова здібність** — одна активована здібність на кулдауні (`8 год`), унікальна для
   архетипу. Кнопка на картці.
4. **Підсилення емоцій** — 🎬 Драма множить силу емоцій (`emoMult`), решта = 1.0.

Пасив/синергія застосовуються до **поточного домінантного** архетипу — тож поки характер
дрейфує (стадії 🫠/👾), білд змінюється в реальному часі.

## Дані (розширення `PERSONALITIES` у `personalities.js`)

Кожен архетип отримує `traits`:
```js
traits: {
  decay:    { <stat>: множник, ... },   // пасив занепаду (відсутнє = 1.0)
  likes:    ["play", ...],               // дії-резонанс (×1.4 + 💖)
  dislikes: ["clean", ...],              // дії-бекфайр (×0.6, −5 mood + 😤)
  emoMult:  1.0,                         // підсилення емоцій (drama=1.3)
  ability:  { label, toast, effects:{...} | rand:true } // фірмова здібність
}
```

Розклад (стисло):

| Архетип | decay | likes | dislikes | ability |
|---|---|---|---|---|
| 😈 Ґремлін | hygiene ×1.3 | play | clean | 😈 Капость (хаос: рандомні ±) |
| 🌌 Філософ | sanity ×0.7, purpose ×0.7 | meditate, calm | — | 🌌 Прозріння (+purpose+sanity) |
| 🌞 Сонечко | mood ×0.6 | play, talk | — | 🌞 Промінчик (+mood+social) |
| 🎬 Драма | mood ×1.1 · emoMult 1.3 | talk | sleep | 🎬 Сцена (+mood, −energy) |
| 😴 Сонько | energy ×0.65, mood ×1.15 | sleep | play | 😴 Глибокий сон (+energy, −mood) |
| 🤓 Ботанік | hygiene ×0.7 | clean | — | 🤓 Вивчити (+sanity) |
| 😎 Бунтар | warmth ×0.85, social ×0.8 | — | calm | 😎 Сам по собі (+warmth+social) |

## Зміни в `pet.js`

- Нове поле `last_ability` (конструктор + `toJSON`); константа `ABILITY_COOLDOWN_H = 8`.
- Хелпери: `_traits()` (traits домінантного архетипу або `{}`), `_decayMult(stat)`,
  `_actionSynergy(action)` (`'like'|'dislike'|null`), `_chaosEffects()` (рандом для Ґремліна).
- `applyDecay`: занепад кожного активного показника × `_decayMult(stat)`.
- `doAction`: множить основний показник на синергію (×1.4/×0.6); бекфайр додає `−5 mood`;
  **повертає `{ synergy }`** (для флейвору в `index.js`).
- `applyEmotion`: помножити масштаб на `_traits().emoMult`.
- `abilityInfo()` (def або `null` для яйця/без persona), `abilityReadyIn()` (сек до готовності),
  `useAbility()` (перевіряє живий/кулдаун; ефекти лише на активні показники; ставить
  `last_ability`, `counts.ability++`; повертає `{ ok, info, applied }` або `{ ok:false, reason }`).
- `personaPerks()` — короткий HTML-блок (здібність + смаки) для `/personality`.

## Зміни в `index.js`

- `mainKeyboard`: для не-яйця/живого додаємо кнопку здібності (callback `abil`) у ряд із «Подія/Ріст».
- `handleCallback` `abil`: кулдаун → модалка «ще відпочиває (~N год)»; інакше `useAbility` →
  тост + дельти показників (як у подіях) + можливе досягнення; тиха перерисовка картки.
- Обидва місця виклику `doAction`: додаємо флейвор синергії до тоста (💖 / 😤).
- `/personality`: дописуємо `personaPerks()`.

## Тести (`test/persona-mechanics.test.mjs`, без фреймворку)
- `_decayMult`: множники по архетипах + дефолт 1.0.
- `doAction`: like (×1.4, synergy='like'), dislike (×0.6 + −5 mood, 'dislike'), нейтрально (null).
  (Перед викликом `last_update=now()` → занепад no-op, дельти детерміновані.)
- `applyEmotion`: 🎬 Драма дає більший приріст, ніж 🤓 Ботанік за тих самих вдачі/сили.
- `abilityInfo`/`abilityReadyIn`/`useAbility`: є для архетипу / null для яйця; ефекти лише
  на активні показники; кулдаун після використання; мертвий → `ok:false`.
- round-trip `toJSON`: `last_ability` зберігається.

## Межі / баланс
- Здібності — **помірні** (трейд-офи або середні бусти, не «→100»), кулдаун 8 год — щоб не
  знецінити основний цикл догляду.
- Пасиви помірні (×0.6…×1.3), не ламають смерть/виживання.
- Яйце/без-persona: пасивів/синергії/здібності немає (домінанта = null → `_traits()={}`).
