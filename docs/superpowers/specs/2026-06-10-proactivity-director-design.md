# «Режисер» проактивності аватара

**Дата:** 2026-06-10
**Статус:** затверджено власником проєкту (повна автономія реалізації)

## Мета

Зараз cron-нагадування плаского типу: або «мені бракує X», або «нудно». Через
одноманітність людина швидко звикає. Замість збільшення частоти/вартості — на той
самий слот, коли cron вирішив вийти на звʼязок, **обираємо ТИП виходу** за контекстом.
Це дає різноманіття без зростання кількості LLM-викликів (одне повідомлення на тік на пета).

## Типи виходу на звʼязок

Два рівні пріоритетності:

**Пріоритетні** (без кидка кубика, раз на день — обходять звичайні умови потреби/тиші):
- `morning` — ранкове привітання у характері. Лише для **активних за останні 48 год**
  (щоб не діймати тих, хто давно зник — їм лишається звичайний silence-нудж). LLM.
- `streak` — порятунок денної серії: була серія ≥2, активний учора, але **не сьогодні**,
  і зараз день/вечір. **Без LLM** (шаблон). Рятує retention.

**Флейворні** (лише коли вже є привід вийти — гостра потреба або тиша ≥4 год; з шансом):
- `emotion` — досі під враженням сильної емоції (intensity ≥ 4).
- `need` — є гостра потреба (`worstNeed`). Класичний нудж.
- `memory` — згадка конкретного факту з памʼяті («ну як той іспит?»).
- `thought` — спонтанна абсурдна думка/«відкриття», без скарг. **Завжди можливий** фолбек.

Серед флейворних `tick` обирає **випадково** (різноманіття); серед пріоритетних — перший.

## Розподіл відповідальності

- **`pet.proactivePlan(dayPart, today)`** — ЧИСТА функція. Повертає
  `{ priority: [...], flavor: [...] }` — які типи доречні за поточним станом. Тестується.
- **`pet.streakRescueLine()`** — шаблонний рядок порятунку серії (без LLM). Тестується.
- **`tick` (index.js)** — вся випадковість (шанс, вибір флейвору), кулдаун, надсилання,
  оновлення `rituals`/`last_nudge`. I/O-glue, не тестується юнітами.
- **`brain.nudge(env, pet, dayPart, kind)`** — будує user-prompt під тип (`NUDGE_PROMPTS`);
  системний промпт уже несе памʼять/емоцію/стан. `streak` сюди не йде (шаблон у pet.js).

## Модель даних (нове в `Pet`)

```jsonc
"rituals": { "morning": <dayIdx>, "streak": <dayIdx> }  // коли востаннє відпрацював ритуал
```

`dayIdx` = `floor((unix + tz*3600) / 86400)` — той самий день-індекс, що й у `registerActivity`.

## `proactivePlan` — правила (псевдо)

```
if (isEgg || !alive) → { priority:[], flavor:[] }
priority:
  morning  ← dayPart=morning  & seen<48год  & rituals.morning≠today
  streak   ← streak≥2 & last_active_day=today-1 & dayPart∈{day,evening} & rituals.streak≠today
flavor:
  emotion  ← emotion активна & intensity≥4
  need     ← worstNeed≠null
  memory   ← є memories
  thought  ← завжди
```

## Зміни в `tick`

1. Яйце — окремий простий шлях (тизери без LLM), як і було (винесено в гілку циклу).
2. Для тих, хто вилупився: `plan = proactivePlan(dp, today)`.
   - `priority` непорожній → `kind = priority[0]` (без кидка кубика).
   - інакше: лишається стара умова `needsAttention || longSilence` + `chance` → `kind` випадково з `flavor`.
3. `kind==="streak"` → `pet.streakRescueLine()`; інакше `brain.nudge(env, pet, dp, kind)`.
4. Після надсилання: `last_nudge=now()`; для `morning`/`streak` — `rituals[kind]=today`.
5. Нічна тиша й кулдаун 2 год — без змін.

## Межі / вартість
- **Одне повідомлення на тік на пета** — вартість не зростає з типами.
- Ранкове привітання й порятунок серії — **раз на день** (guard через `rituals`), тож без спаму.
- Лапснутим (>48 год тиші) ранкові привітання НЕ шлемо — лише звичайний silence-нудж.
- `thought` завжди у флейворі → випадковий вибір безпечний (масив непорожній).
- Зіпсована LLM-відповідь → `brain.nudge` має фолбек (як і раніше).

## Тести (`test/proactivity.test.mjs`, без фреймворку)
- `proactivePlan`: egg/dead → порожньо; morning (є/greeted/лапснутий/не-ранок);
  streak (під загрозою / активний сьогодні / серія<2 / запінгований / не день);
  flavor: emotion (сильна/слабка), need, memory, thought-завжди.
- `streakRescueLine`: непорожній рядок із числом серії.
