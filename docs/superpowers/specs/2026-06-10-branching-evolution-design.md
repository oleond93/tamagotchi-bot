# Гілчаста еволюція (Digimon-style) — Фаза 2

**Дата:** 2026-06-10
**Статус:** затверджено власником (повна автономія). Фаза 2 із двох.

## Мета

Зробити **вдачу × характер** механічно вагомими через еволюцію: фінальна форма
**гілкується** залежно від того, ким став улюбленець. Раптом раннє формування має
велику вагу й реіграбельність («а що, як ростити холодно й хаотично?»).

## Принцип (мінімальний ризик)

**Часовий кістяк еволюції НЕ чіпаємо:** вік→індекс, активні показники (`STAGE_STATS`),
стиль мовлення за зрілістю — усе лишається index-driven. Гілкується лише **ІДЕНТИЧНІСТЬ
форми** (назва + емодзі + опис) — як «вид» при тому самому «рівні» у Digimon.

**Гілкуємо лише фінальну форму** (index 5, 30 днів) — це памʼятна кульмінація («ким воно
стало»). Проміжні стадії спільні. Гілкування проміжних — можлива майбутня ітерація.

До фінальної форми персона вже застигла (index 3) і вдача фіксована (вилуплення) — шлях
стабільний на момент фіналу. На ранніх стадіях картка росту **тизерить** майбутню форму
(і вона може зсунутись, поки характер дрейфує — тематично).

## Шлях (`evolutionPath`)

Чиста функція (вдача × домінантний характер) → ключ форми (або `null` без persona):

| Характер | Світла вдача (😇/🥰) | Темна вдача (😈/😼) | Нейтрально |
|---|---|---|---|
| 😈 Ґремлін | 🃏 Дух-Бешкетник (`imp`) | 👹 Хаос-Демон (`demon`) | imp |
| 😎 Бунтар | 🦸 Вільний Дух (`free`) | 🦹 Відступник (`outlaw`) | free |
| 🌞 Сонечко | 🌟 Світлий Янгол (`angel`) | ☀️ Сонцесяйний (`sun`) | sun |
| 🌌 Філософ | 🧘 Просвітлений (`enlightened`) | 🌌 Космічний Розум (`void`) | void |
| 🎬 Драма | 🌠 Наднова-Зірка (`star`) | | |
| 🤓 Ботанік | 🛸 Розумна Машина (`machine`) | | |
| 😴 Сонько | 💤 Сон-Сутність (`dream`) | | |
| (без persona) | 🌌 Космічна Сутність (`cosmic`, дефолт) | | |

«Світла» = angelic/gentle; «темна» = wild/feisty; «нейтрально» = balanced (та решта).

## Зміни в `pet.js`

- Дані: `FORMS` (ключ → {emoji, name, desc}); `FINAL_INDEX = EVOLUTION_STAGES.length-1`.
- `evolutionPath()` — ключ форми за вдачею×характером (або null).
- `_stageIdentity(index)` — {name, emoji, desc}: для `index === FINAL_INDEX` із наявним
  шляхом → `FORMS[path]`; інакше канонічний `EVOLUTION_STAGES[index]`. Для не-фінальних
  індексів **не** обчислює шлях (нульовий оверхед — викликається часто).
- `stage()`, `evolutionInfo()` (current+next), `growthCard()` — через `_stageIdentity`.
- `celebrationFor(FINAL_INDEX)` — особливе «ФІНАЛЬНА ЕВОЛЮЦІЯ»: оголошує вид і
  **пояснює, що його сформували саме вдача + характер** (хук реіграбельності).
- LLM-промпт (`stateContext`) автоматично отримує нову назву стадії через `stage()`.

## Сумісність
- Наявні пети на 30+ днях із persona одразу показуватимуть гілчасту форму (це і є фіча).
- `seen_stage` не дає повторних святкувань заднім числом — без спаму.
- `achievements.js` не залежить від назв стадій — без змін.

## Тести (`test/evolution.test.mjs`, без фреймворку)
- `evolutionPath`: gremlin×wild→demon, gremlin×gentle→imp; sunshine×angelic→angel,
  sunshine×balanced→sun; philosopher×angelic→enlightened, philosopher×wild→void;
  rebel×wild→outlaw, rebel×gentle→free; drama→star, nerd→machine, sleepy→dream;
  яйце/без persona→null.
- `_stageIdentity`: фінал із gremlin×wild → «Хаос-Демон»/👹; проміжний (index2) → канон;
  фінал без persona → «Космічна Сутність».
- `stage()` на 31 дні з gremlin×wild → «Хаос-Демон».
- `evolutionInfo().next` на index4 (15 дн) з drama → тизер «Наднова-Зірка».
- `celebrationFor(FINAL_INDEX)` містить назву форми та згадку вдачі/характеру.

## Поза скоупом (YAGNI)
Гілкування проміжних стадій, окремий стиль мовлення per-form, нова «ачівка» за фінал,
візуальні скіни картки per-form — наступні ітерації.
