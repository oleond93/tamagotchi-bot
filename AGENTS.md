# AGENTS.md — tamagotchi-bot

Гайд для AI-агентів (і людей), що працюють над цим проєктом. Описує стан, архітектуру й правила.

> ⚠️ **ГОЛОВНЕ ПРАВИЛО:** при **будь-якій зміні ігрових механік чи лору** обов'язково онови `game-schema.html` (схема для розробників) **і `src/guide-page.js`** (повний веб-гайд для гравців на `/guide`), щоб вони лишались консистентними з кодом. Деталі — у розділі [«Документ-схема гри»](#-документ-схема-гри).

---

## 🎯 Що це

**tamagotchi-bot** — абсурдно-фановий віртуальний улюбленець у Телеграмі (українською). Один спільний бот для всіх: розгортається **один раз**, кожен Telegram-чат отримує власного тамагочі. Користувачі нічого не встановлюють — просто пишуть боту.

**Жорстке обмеження — вартість ≈ $0.** Усе має лишатися в межах безкоштовних тарифів (Cloudflare Workers + D1 + cron, безкоштовні LLM-моделі). Не додавати платних залежностей чи важких фреймворків.

Мова всього (код-коментарі, контент бота, спілкування з власником проєкту) — **українська**.

---

## 🏗️ Архітектура й стек

- **Cloudflare Worker** — один воркер, дві точки входу (`src/index.js`):
  - `fetch` — webhook Телеграму (`POST /tg/<TELEGRAM_TOKEN>`) + службові ендпоінти `/init` (разове налаштування вебхука), `/diag` (діагностика LLM) і `/guide` (повний веб-гайд по грі, аліас `/gide`).
  - `scheduled` — cron `0 * * * *` (щогодини): занепад стану + проактивні нагадування.
- **D1 (SQLite)** — таблиця `pets(chat_id INTEGER PRIMARY KEY, data TEXT)`. Увесь стан пета зберігається як JSON у `data` (див. `Pet.toJSON()`). Схема створюється автоматично (`ensureSchema`), міграції вручну не потрібні.
- **LLM** — будь-який OpenAI-сумісний chat/completions API. Дефолт OpenRouter (`openrouter/free`). Пріоритет змінних: `LLM_*` > `OPENROUTER_*` > фолбек.
- Без фреймворків, чистий ES-модуль (`"type": "module"`).

### Конфіг і секрети
- `wrangler.toml`: `keep_vars = true` (деплой **не** затирає змінні/секрети з дашборду), cron, біндинг D1 з реальним `database_id`.
- Секрети (в дашборді Cloudflare, не в репо): `TELEGRAM_TOKEN`, `OPENROUTER_API_KEY` (або `LLM_API_KEY`).
- Опційні змінні: `OPENROUTER_MODEL`/`LLM_MODEL`, `LLM_BASE_URL`, `TZ_OFFSET_HOURS` (типово 3 — Київ).

### Деплой
Репо: `git@github.com:oleond93/tamagotchi-bot.git`. Кнопка «Deploy to Cloudflare» у README, або CLI: `npm install` → `npx wrangler d1 create tamagotchi` → `npx wrangler deploy` → додати секрети → відкрити `/init`.

## 🎮 Ігрові механіки (стисло)

> Повний, завжди актуальний опис із діаграмою й таблицями — у **`game-schema.html`**. Тут — короткий конспект.

- **8 показників:** 🔥 warmth, 🍗 hunger, 🛁 hygiene, ⚡ energy, 🎉 mood, 🤪 sanity, 🫂 social, 🌌 purpose. Відкриваються поступово зі стадіями (`STAGE_STATS`).
- **Занепад** (`DECAY_PER_HOUR`): агресивний, 100→0 за ~1.5–2.9 год. Уночі (22:00–08:00 локального часу) множиться на `NIGHT_FACTOR = 0.18` (≈5.5× повільніше), щоб пет пережив сон.
- **6 стадій еволюції за віком (днів):** 🥚 Яйце (0) → 🫠 Слизька Грудка (1) → 👾 Дивне Створіння (3) → 😈 Хаотичний Монстр (7) → 🐉 Стародавнє Зло (14) → 🌌 Космічна Сутність (30). Кожна стадія: свій набір показників, кнопок, подій, стилю мовлення LLM. Переходи святкуються повідомленням.
- **8 дій** (`ACTIONS`): кожна відновлює свій показник із побічними ефектами. Доступні лише дії активних показників стадії.
- **Вдача (temperament, 5 град.):** фіксується **назавжди при вилупленні** за середнім теплом яйця (`egg_warm_sum/egg_hours`): 😇 angelic → 🥰 gentle → 😌 balanced → 😼 feisty → 😈 wild.
- **Характер (persona, 7 архетипів):** при вилупленні **випадковий**; перші 2 стадії (🫠, 👾) **дрейфує** від догляду (`ACTION_AFFINITY` + чат/події), може стати **сумішшю двох** (≥60%); на стадії 😈 (index 3) **застигає** (`persona_locked`). Персона і вдача — різні осі.
- **LLM-промпт** = характер + вдача + стиль за стадією + пора доби + поточний стан.
- **Події:** 60 (по 10/стадія), вибір із 2 варіантів, ефекти лише на активні показники, кулдаун 2 год, результат у модалці. Деякі вибори ставлять прапорці для досягнень.
- **23 досягнення:** на основі лічильників/прапорців/стріку/вдачі/віку.
- **Смерть:** лише з стадії 😈+ (коли є sanity), коли hunger=0 і sanity=0 одночасно. `/revive` повертає активні показники до 60%.
- **Нагадування (cron):** щогодини вдень (уночі тиша), кулдаун 2 год, спрацьовує при гострій потребі або тиші ≥4 год. Яйце нагадує частіше.

## 🖥️ UX-конвенції

- Основний інтерфейс — **inline-кнопкова картка** (`statusCard` + `mainKeyboard`), кнопки генеруються під поточну стадію.
- **Постійна клавіатура** (`petMenu`, reply keyboard біля поля вводу) з кнопками `📊 Картка` / `📖 Гайд` — щоб inline-картку можна було повернути одним тапом, навіть коли вона поїхала вгору під час чату. Кнопки шлють текст, який мапиться на `/status` / `/guide`. Telegram показує reply-клавіатуру лише з повідомленням, що її несе, а картка несе inline-клавіатуру — тож `ensureMenu()` **одноразово** (прапорець `pet.menu_set`) надсилає окреме повідомлення з `petMenu`, аби клавіатура з'явилась навіть тим, хто лише тисне кнопки. Викликається з `getPet`, `handleCallback` і `/start`.
- **Усі реакції — у модальному вікні** (`answerCallbackQuery` з `show_alert: true`): дії, егг-взаємодії (listen/wiggle), події, досягнення. Маленька спливна підказка зверху (toast) не використовується для змістовних реакцій — її погано видно.
- Картка позаду тихо оновлюється через `editMessageText`; «message is not modified» глушиться `try/catch`.
- Картка ЯЙЦЯ — особлива (акцент на вилупленні + «передчуття»). Гайд без спойлерів фіналу.

## 📐 Документ-схема гри

`game-schema.html` — самодостатній HTML (Mermaid-діаграма зв'язків + таблиці), **згенерований зі стану коду**. Це канонічне джерело правди про механіки для людей.

> 🔴 **ОБОВ'ЯЗКОВО:** змінюючи будь-яку механіку — швидкості занепаду, ефекти дій, пороги стадій/вдачі, набір показників/дій/подій/досягнень, таймери, формули — **синхронно онови `game-schema.html`** (відповідні таблиці, діаграму, легенди). Розсинхрон коду й схеми неприпустимий. Якщо зміна суттєва — також онови `README.md` і пам'ять (`game-design.md`).

### 📖 Повний веб-гайд (`src/guide-page.js`, маршрут `/guide`)

`src/guide-page.js` — це **канонічний гайд для ГРАВЦІВ** (самодостатня HTML-сторінка): лор + усі механіки, по категоріях, з навігацією. Експортує `GUIDE_HTML`; `index.js` віддає його на `/guide` (аліас `/gide`).

> 🔴🔴 **ЗАЛІЗНЕ ПРАВИЛО:** **БУДЬ-ЯКА зміна — навіть найдрібніша — у механіках АБО лорі гри має бути відображена в цьому гайді.** Жодних винятків. Сюди входить геть усе:
> - **Механіки:** показники, швидкості занепаду, ефекти/синергія дій, стадії та їх активні показники, гілчасті форми (вдача × характер), пороги/множники вдачі, архетипи характеру та їх дрейф/пасиви/здібності, емоції, памʼять, проактивність (типи нагадувань, таймери), події, досягнення, смерть/воскресіння, стрік, час доби, команди й кнопки, будь-які числа/формули/кулдауни.
> - **Лор:** опис світу, премиса, тон, назви/описи форм і архетипів, тексти стадій, будь-які наративні деталі.
>
> Додаєш фічу — додаєш розділ (і пункт у сайдбар-навігації). Змінюєш число — правиш його в гайді. Міняєш текст лору — синхронізуєш формулювання. **Розсинхрон коду й гайда неприпустимий.**
>
> Після правок бажано перевірити рендер (значення/таблиці/навігація) — напр. вивантажити `GUIDE_HTML` у файл і відкрити в браузері.

## ✅ Чек-ліст при зміні механік

1. Зміни у `src/` (зазвичай константи в `pet.js` / `index.js`).
2. **Онови `game-schema.html`** (таблиці/діаграма/легенди) **і `src/guide-page.js`** (повний веб-гайд `/guide`).
3. За потреби онови `README.md` («Що вміє бот») і `guide.js` (якщо змінився досвід гравця).
4. Перевір через `wrangler dev` / бот.
5. Коміт (українською в тілі за бажанням; футер `Co-Authored-By` за конвенцією).
